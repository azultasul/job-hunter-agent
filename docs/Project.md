# Job-Hunter-Agent Web Service

본 문서는 **Job-Hunter-Agent**를 기반으로 한 웹 서비스의 약식 PRD(Product Requirements Document)입니다. 서비스의 목적, 사용자 흐름, 주요 기능, 기술 스택, 시스템 구성 등을 간략하지만 구현 관점에서 명확히 정리하는 것을 목표로 합니다.

---

## 1. 서비스 개요

### 1.1 목적

Job-Hunter-Agent Web Service는 구직자가 **채용 공고 탐색 → 직무 매칭 → 이력서 최적화 → 기업 조사 및 면접 준비 → 결과물 요약 및 다운로드**까지의 과정을 하나의 흐름으로 수행할 수 있도록 돕는 AI 기반 취업 지원 서비스입니다.

기존의 Job-Hunter-Agent를 백엔드 에이전트 레벨에서 확장하여, 실제 사용자 인터랙션이 가능한 웹 서비스 형태로 제공합니다.

### 1.2 타겟 사용자

- 주니어/미들 개발자
- 다수의 채용 공고에 지원해야 하는 구직자
- 이력서/면접 준비를 체계적으로 자동화하고 싶은 사용자

---

## 2. 전체 사용자 플로우

1. **첫 화면 (입력 단계)**

   - 사용자가 희망 조건을 입력
   - 사용자 이력서 업로드
   - 확인 버튼 클릭
   - 백엔드에서 채용 공고 검색 에이전트 + 직무 매칭 에이전트 실행

2. **두 번째 화면 (결과 및 작업 단계)**

   - 탭 기반 UI
   - 각 탭은 Job-Hunter-Agent의 주요 단계에 대응

---

## 3. 화면 및 기능 상세

### 3.1 첫 화면: 조건 입력 및 이력서 업로드

#### 주요 기능

- 희망 조건 입력

  - 직무
  - 기술 스택
  - 근무 형태 (예: 정규직, 원격 등)
  - 기타 필터 조건

- 이력서 업로드

  - PDF / DOCX 등

- 확인 버튼 클릭 시 처리 흐름

  - 채용 공고 검색 에이전트 실행 (크롤링)
  - 직무 매칭 에이전트 실행
  - 결과(output 1) 생성

---

### 3.2 두 번째 화면: 탭 기반 결과 화면

상단에는 **서비스 단계 탭**이 존재하며, 각 탭은 독립적인 결과와 상호작용을 제공합니다.

---

### 3.2.1 첫 번째 탭: 최적 매칭 결과 (Output 1)

#### 설명

- 1번 단계에서 생성된 **최적 매칭 결과**를 시각적으로 표시

#### 주요 UI 요소

- 매칭된 채용 공고 리스트
- 각 공고에 대한 매칭 점수
- 주요 매칭 사유 요약

#### 데이터 출처

- 채용 공고 검색 에이전트
- 직무 매칭 에이전트

---

### 3.2.2 두 번째 탭: 이력서 최적화 (Output 2)

#### 설명

- 최적 매칭 결과를 기준으로 이력서 최적화 에이전트 실행
- 맞춤형 이력서 생성

#### 주요 UI 요소

- 좌측: 기존 이력서
- 우측: AI가 수정한 이력서
- 변경 사항 diff 표시

  - 추가 / 수정 / 삭제 구분

#### 핵심 포인트

- 사용자는 변경 내용을 직관적으로 비교 가능
- 추후 수동 수정 확장 가능성 고려

---

### 3.2.3 세 번째 탭: 기업 조사 & 면접 준비 (Output 3)

#### 설명

- 최적 매칭 결과 기반으로

  - 기업 조사 에이전트 실행
  - 면접 준비 에이전트 실행

#### 주요 UI 요소

- 좌측 영역: 기업 정보

  - 기업 개요
  - 주요 사업
  - 기술 스택
  - 최근 이슈 (가능한 경우)

- 우측 영역: 면접 대비 문서

  - 예상 질문
  - 기술 질문
  - 답변 가이드

#### 특징

- 면접 대비 문서는 **실시간 사용자 수정 가능**
- 편집 가능한 문서 형태로 제공

---

### 3.2.4 네 번째 탭: 요약 및 다운로드

#### 설명

- 이전 단계에서 생성된 결과물을 한 번에 관리하는 요약 탭

#### 주요 UI 요소

- 생성된 문서 목록

  - Output 1: 최적 매칭 결과
  - Output 2: 맞춤형 이력서
  - Output 3: 면접 대비 문서

- PDF 다운로드 버튼

#### 기능

- 선택 또는 전체 문서 PDF 변환
- 최종 제출용 결과물 생성

---

## 4. Output 문서 형식 및 표현 방식

### 4.1 Output 문서 공통 규칙

본 서비스에서 생성되는 모든 결과물(Output 1, 2, 3)은 **Markdown(md) 형식**을 기준으로 생성·관리됩니다.

#### 적용 대상

- Output 1: 최적 매칭 결과 문서
- Output 2: 맞춤형 이력서
- Output 3: 기업 조사 및 면접 대비 문서

#### 설계 의도

- 문서 구조의 일관성 유지
- 에이전트 결과물의 가독성 및 재사용성 확보
- PDF 변환, 버전 관리, diff 비교에 용이

### 4.2 프론트엔드 표현 방식

- 프론트엔드에서는 모든 Output 문서를 **Markdown 렌더링** 방식으로 화면에 표시
- Markdown → HTML 변환을 통해 다음 기능을 지원

  - 제목, 리스트, 코드 블록, 강조 표현
  - diff 기반 비교 UI (이력서 최적화 탭)

- 면접 대비 문서(Output 3)는

  - Markdown 기반 **실시간 편집 가능 문서**로 제공

### 4.3 다운로드 및 변환

- 최종 요약 탭에서 Markdown 문서를 기반으로

  - PDF 변환 및 다운로드 지원

- Markdown을 단일 소스(Single Source of Truth)로 사용
